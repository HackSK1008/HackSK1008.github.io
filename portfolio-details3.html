Project Overview

Saheli is an AI-driven safety app for women that detects distress, sends instant alerts, and shares real-time location with emergency contacts and authorities. Designed for discreet activation and rapid response.

Key Features
  • Voice-Activated SOS: Detects the keyword and triggers emergency alerts without physical phone interaction.
  • Proximity-Based Alerts: Notifies nearest contacts based on geolocation to maximize response speed.
  • Fake Shutdown & Call: Disguises the device while critical functions run in the background for safety.
  • Police Dashboard: Real-time incident dashboard for authorities with mapping and alert management.
Technologies

Python, JavaScript, Google Maps API, Twilio, Cloud Backend, Mobile (Flutter)

Outcome

Improved response time and better situational awareness for emergency services; recognized at multiple hackathons for real-world impact.

Technical Summary

Saheli integrates machine learning-based voice detection with geolocation and cloud analytics to create an end-to-end safety system. It prioritizes discreet activation, low-latency alerting, and privacy-preserving data handling.

System Architecture
  1. Local audio capture → Lightweight keyword detection model → Trigger event
  2. Event → Location fetch → Alert composition (contacts + police)
  3. Server-side ingestion → Dashboard visualization & alert escalation
Implementation Notes
  • Keyword detection uses an amplitude + spectral feature based ML classifier tuned for low false positives.
  • Fake-shutdown preserves background service via foreground service pattern (Android) and periodic heartbeats to server.
  • Proximity is calculated using Haversine formula on contact coordinates, filtering contacts by distance and availability.
Results & Metrics
  • Successful keyword detection rate (controlled tests): ~92%
  • Average alert dispatch latency: < 6 seconds
  • Pilot user adherence and satisfaction showed measurable improvement across trials.
